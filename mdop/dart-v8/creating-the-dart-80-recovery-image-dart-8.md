# DaRT 8.0 복구 이미지 만들기


Microsoft 진단 및 복구 도구 집합 (DaRT) 8.0을 설치한 후 DaRT 8.0 복구 이미지를 만듭니다. 복구 이미지는 Windows RE를 시작 하 여 DaRT 도구를 시작할 수 있습니다. ISO (국제 표준화 기구) 파일과 WIM (Windows Imaging 형식) 이미지를 생성할 수 있습니다. 또한 PowerShell을 사용 하 여 DaRT 복구 이미지 마법사에서 선택한 설정을 사용 하는 스크립트를 생성할 수 있습니다. 나중에이 스크립트를 사용 하 여 동일한 설정을 사용 하 여 복구 이미지를 다시 작성할 수 있습니다. 복구 이미지는 다양 한 복구 도구를 제공 합니다. 도구에 대 한 설명은 [DaRT 8.0의 도구 개요](overview-of-the-tools-in-dart-80-dart-8.md)를 참조 하세요.

컴퓨터를 DaRT로 부팅 한 후 다른 DaRT 도구를 실행 하 여 컴퓨터를 진단 하 고 복구 해 볼 수 있습니다. 이 섹션에서는 DaRT 복구 이미지를 만드는 과정을 안내 하 고 이미지의 일부로 포함 하려는 도구와 기능을 선택할 수 있습니다.

다음 두 가지 방법 중 하나를 사용 하 여 DaRT 복구 이미지를 만들 수 있습니다.

-   Windows 환경에서 실행 되는 DaRT 복구 이미지 마법사를 사용 합니다.

-   원하는 값을 사용 하 여 예제 PowerShell 스크립트를 수정 합니다. 자세한 내용은 [PowerShell 스크립트를 사용 하 여 복구 이미지를 만드는 방법을](how-to-use-a-powershell-script-to-create-the-recovery-image-dart-8.md)참조 하세요.

기록 가능한 CD 또는 DVD에 ISO를 작성 하 고, USB 플래시 드라이브에 저장 하거나, 원격 파티션이나 복구 파티션에서 DaRT로 부팅 하는 데 사용할 수 있는 형식으로 저장할 수 있습니다.

ISO 이미지를 만들었으면 빈 CD 또는 DVD (컴퓨터에 CD 또는 DVD 드라이브가 있는 경우)로 구울 수 있습니다. 컴퓨터에이 용도의 드라이브가 없는 경우 Cd 또는 Dvd를 굽는 데 사용 되는 대부분의 일반적인 프로그램을 사용할 수 있습니다.

## 이미지 아키텍처를 선택 하 고 경로를 지정 합니다.


Windows 8 미디어 페이지에서 32 비트 또는 64 비트 DaRT 복구 이미지를 만들지 여부를 선택 합니다. 32 비트 Windows를 사용 하 여 32 비트 DaRT 복구 이미지 및 64 비트 Windows를 작성 하 여 64 비트 DaRT 복구 이미지를 작성 합니다. 단일 컴퓨터를 사용 하 여 두 아키텍처 유형 모두에 대 한 복구 이미지를 만들 수 있지만, 32 비트 및 64 비트 아키텍처 모두에서 작동 하는 하나의 이미지를 만들 수는 없습니다. 또한 Windows 8 설치 미디어의 경로도 표시 합니다. 만들고 있는 복구 이미지 중 하 나와 일치 하는 아키텍처를 선택 합니다.

**이미지 아키텍처를 선택 하 고 경로를 지정 하려면**

1.  **Windows 8 미디어** 페이지에서 다음 중 하나를 선택 합니다.

    -   64 비트 컴퓨터에 대 한 복구 이미지를 만드는 경우 **x64 (64 비트) DaRT 이미지 만들기**를 선택 합니다.

    -   32 비트 컴퓨터에 대 한 복구 이미지를 만드는 경우 **x86 (32 비트) DaRT 이미지 만들기**를 선택 합니다.

2.  **Windows 8 &lt; 64 비트 또는 32 비트 &gt; 설치 미디어의 루트 경로 지정** 상자에 windows 8 설치 파일의 경로를 입력 합니다. 만들고 있는 복구 이미지의 아키텍처와 일치 하는 경로를 사용 합니다.

3.  **다음**을 클릭합니다.

## 복구 이미지에 포함할 도구 선택


도구 페이지에서 복구 이미지에 포함할 여러 도구를 선택할 수 있습니다. 이러한 도구는 최종 사용자가 DaRT 이미지로 부팅 하는 경우에 사용할 수 있습니다. 그러나 DaRT 이미지를 만들 때 원격 연결을 사용 하도록 설정 하면 이미지에 포함 하도록 선택한 도구에 관계 없이 지원 센터 작업자에 게 최종 사용자의 컴퓨터에 연결 하는 경우 모든 도구를 사용할 수 있습니다.

이러한 도구에 대 한 최종 사용자의 액세스를 제한 하지만 원격 연결 뷰어를 통해 도구에 대 한 전체 액세스는 유지 하려면 도구 페이지에서 해당 도구를 선택 하지 마세요. 최종 사용자는 원격 연결만을 사용할 수 있으며, 복구 이미지에서 제외 된 도구는 볼 수만 있고 액세스는 하지 못할 수도 있습니다.

**복구 이미지에 포함할 도구를 선택 하려면**

1.  **도구** 페이지에서 이미지에 포함 하려는 각 도구 옆에 있는 확인란을 선택 합니다.

2.  **다음**을 클릭합니다.

## 헬프 데스크에의 한 원격 연결 허용 여부 선택


원격 연결 페이지에서 지원 센터 작업자에 게 원격으로 연결 하 여 최종 사용자의 컴퓨터에 있는 DaRT 도구를 실행할 수 있도록 선택 합니다. 그러면 원격 연결 옵션이 진단 및 복구 도구 집합 창에서 사용할 수 있는 옵션으로 표시 됩니다. 지원 센터 작업자는 원격 연결을 설정 하 고 나면 원격 위치에서 최종 사용자 컴퓨터에 있는 DaRT 도구를 실행할 수 있습니다.

**지원 센터 근로자에의 한 원격 연결을 허용할지 여부를 선택 하려면**

1.  **원격 연결** 페이지에서 원격 연결 **허용** 확인란을 선택 하 여 원격 연결을 허용 하거나 확인란의 선택을 취소 하 여 원격 연결을 방지 합니다.

2.  **원격 연결 허용** 확인란의 선택을 취소 한 경우 **다음**을 클릭 합니다. 그렇지 않으면 다음 단계로 이동 하 여 원격 연결 구성을 계속 합니다.

3.  다음 중 하나를 선택하세요.

    -   Windows에서 열려 있는 포트 번호를 선택 하도록 합니다.

    -   포트 번호를 지정 합니다. 이 옵션을 선택 하는 경우 옵션 아래에 있는 필드에 1과 65535 사이의 포트 번호를 입력 합니다. 이 포트 번호는 원격 연결을 설정할 때 사용 됩니다. 충돌의 가능성을 최소화 하기 위해 포트 번호를 1024 이상으로 하는 것이 좋습니다.

4.  (선택 사항) **원격 연결 시작** 메시지 상자에서 원격 연결을 설정할 때 최종 사용자가 받는 사용자 지정 메시지를 만듭니다. 메시지는 최대 2048 자를 초과할 수 있습니다.

5.  **다음**을 클릭합니다.

    DaRT 도구를 원격으로 실행 하는 방법에 대 한 자세한 내용은 [Dart 복구 이미지를 사용 하 여 원격 컴퓨터를 복구 하는 방법을](how-to-recover-remote-computers-by-using-the-dart-recovery-image-dart-8.md)참조 하세요.

## 복구 이미지에 드라이버 추가


고급 옵션 페이지의 드라이버 탭에서 컴퓨터를 복구 하는 데 필요할 수 있는 추가 장치 드라이버를 추가할 수 있습니다. 여기에는 일반적으로 Windows 8에서 제공 하지 않는 저장소 또는 네트워크 컨트롤러가 포함 될 수 있습니다. 드라이버는 이미지를 만들 때 설치 됩니다.

**중요**  포함할 드라이버를 선택할 때 무선 연결이 DaRT에서 지원 되지 않는다는 점에 유의 하세요 (예: Bluetooth 또는 802.11 a/b/g/n).

 

**복구 이미지에 드라이버 추가**

1.  **고급 옵션** 페이지에서 **드라이버** 탭을 클릭 합니다.

2.  **추가**를 클릭합니다.

3.  드라이버에 추가할 파일을 찾은 다음 **열기**를 클릭 합니다.

    **참고**  드라이버 파일은 저장소 또는 네트워크 컨트롤러의 제조업체에서 제공 합니다.

     

4.  포함 하려는 모든 드라이버에 대해 2 ~ 3 단계를 반복 합니다.

5.  **다음**을 클릭합니다.

## 복구 이미지에 WinPE 선택적 패키지 추가


고급 옵션 페이지의 WinPE 탭에서 DaRT 이미지에 WinPE 선택적 패키지를 추가할 수 있습니다. 이러한 패키지는 DaRT 복구 이미지 마법사의 설치 조건인 Windows ADK의 일부입니다. 선택할 수 있는 도구는 모두 선택 사항입니다. 필요한 패키지는 도구 페이지에서 선택한 도구에 따라 자동으로 추가 됩니다.

스크래치 공간 크기를 지정할 수도 있습니다. 스크래치 공간은 DaRT를 실행할 수 있도록 별도로 설정 된 RAM 디스크 공간 크기입니다. 스크래치 공간은 최종 사용자의 하드 디스크를 사용할 수 없는 경우에 유용 합니다. 추가 도구 및 드라이버를 실행 하는 경우 스크래치 공간을 늘려야 할 수 있습니다.

**복구 이미지에 WinPE 선택적 패키지 추가**

1.  **고급 옵션** 페이지에서 **WinPE** 탭을 클릭 합니다.

2.  각 패키지 옆에 있는 확인란을 선택 하 여 이미지에 포함 하거나, **이름** 확인란을 클릭 하 여 모든 패키지를 선택 합니다.

3.  **스크래치 공간** 필드에서 최종 사용자의 하드 디스크를 사용할 수 없는 경우에는 DaRT를 실행 하기 위해 할당할 RAM 디스크 공간 크기를 선택 합니다.

4.  **다음**을 클릭합니다.

## 충돌 분석기의 디버깅 도구 추가


ISO 이미지에 충돌 분석기 도구를 포함 하는 경우 Windows 용 디버깅 도구도 포함 해야 합니다. 고급 옵션 페이지의 충돌 분석기 탭에서 충돌 분석기가 메모리 덤프 파일을 분석 하는 데 사용 하는 Windows 8 디버깅 도구 경로를 입력 합니다. DaRT 복구 이미지 마법사를 실행 중인 컴퓨터에 있는 도구를 사용 하거나 최종 사용자 컴퓨터에 있는 도구를 사용할 수 있습니다. 최종 사용자 컴퓨터에서 도구를 사용 하기로 결정 한 경우, 진단 하는 모든 컴퓨터에 디버깅 도구가 설치 되어 있어야 한다는 점에 유의 하세요.

Microsoft Windows SDK (소프트웨어 개발 키트) 또는 WDK (Microsoft Windows 개발 키트)를 설치한 경우 Windows 8 디버깅 도구가 기본적으로 복구 이미지에 추가 되 고 디버깅 도구의 경로는 자동으로 채워집니다. 파일이 기본 파일 경로에 지정 된 위치가 아닌 다른 위치에 있는 경우 Windows 8 디버깅 도구의 경로를 변경할 수 있습니다. 마법사의 링크를 통해 Windows 용 디버깅 도구가 아직 설치 되어 있지 않은 경우이를 다운로드 하 여 설치할 수 있습니다.

Windows 디버깅 도구를 다운로드 하려면 [windows 용 디버깅 도구](https://go.microsoft.com/fwlink/?LinkId=266248)를 참조 하세요. 기본 위치에 디버깅 도구를 설치 합니다.

**참고**  DaRT 마법사는 레지스트리 키의 도구를 확인 `HKLM\Software\Microsoft\Windows Kits\Installed Roots\WindowsDebuggersRoot` 합니다. 레지스트리 값이 없는 경우 마법사는 시스템 아키텍처에 따라 다음 위치 중 하나를 찾습니다.

`%ProgramFilesX86%\Windows Kits\8.0\Debuggers\x64`

`%ProgramFilesX86%\Windows Kits\8.0\Debuggers\x86`

 

**충돌 분석기에 대 한 디버깅 도구를 추가 하려면**

1.  **고급 옵션** 페이지에서 **충돌 분석기** 탭을 클릭 합니다.

2.  ) **디버깅 도구 다운로드** 를 클릭 하 여 Windows 용 디버깅 도구를 다운로드 합니다.

3.  다음 옵션 중 하나를 선택 합니다.

    -   **Windows 8 &lt; 64 비트 또는 32 비트 &gt; 디버깅 도구를 포함**합니다. 이 옵션을 선택 하는 경우 경로가 아직 표시 되지 않은 경우 도구 위치를 찾아 선택 합니다.

    -   **디버그 중인 시스템의 디버깅 도구를 사용**합니다. 이 옵션을 선택 하면 Windows 용 디버깅 도구가 문제 컴퓨터에 없는 경우 충돌 분석기가 작동 하지 않습니다.

4.  **다음**을 클릭합니다.

## Defender 도구에 대 한 정의 추가


고급 옵션 페이지의 Defender 탭에서 Defender 도구에서 사용 하는 정의를 추가 하 여 컴퓨터에서 설치, 실행 또는 변경 하려는 소프트웨어가 원하지 않거나 악의적인 소프트웨어 인지 여부를 확인 합니다.

**Defender 도구에 대 한 정의를 추가 하려면**

1.  **고급 옵션** 페이지에서 **Defender** 탭을 클릭 합니다.

2.  다음 옵션 중 하나를 선택 합니다.

    -   **최신 정의 다운로드 (권장)** – 정의 업데이트가 자동으로 시작 되 고 정의가 DaRT 복구 이미지에 추가 됩니다. 이 옵션은 정의를 사용할 수 없는 경우를 방지 하는 데 도움이 되는 경우에 유용 합니다. 정의를 다운로드 하려면 인터넷에 연결 되어 있어야 합니다.

    -   **나중에 정의 다운로드** – 정의가 dart 복구 이미지에 포함 되지 않으며, dart를 실행 하는 컴퓨터에서 정의를 다운로드 해야 합니다.

        복구 이미지에 최신 정의를 포함 하지 않으려는 경우 또는 복구 이미지에 포함 된 정의가 더 이상 현재 Defender를 사용할 준비가 되어 있지 않은 경우에는 Defender에서 제공 하는 지침에 따라 검색을 시작 하기 전에 최신 정의를 구하십시오.

        **중요**  정의가 없는 경우 스캔할 수 없습니다.

         

3.  **다음**을 클릭합니다.

## 만들 복구 이미지 파일 형식 선택


이미지 만들기 페이지에서 복구 이미지에 대 한 출력 폴더를 선택 하 고, 이미지 이름을 입력 하 고, 만들 DaRT 복구 이미지 파일의 유형을 선택 합니다. 복구 이미지 만들기 프로세스가 진행 되는 동안 Windows 원본 파일의 압축을 풀고, DaRT 파일이이 폴더에 복사 되 고,이 페이지에서 선택한 파일 형식으로 이미지가 "다시 압축" 됩니다.

사용할 수 있는 이미지 파일 형식은 다음과 같습니다.

-   **WIM (Windows Imaging 파일)** -PXE (부팅 실행 환경) 또는 로컬 파티션에 DaRT를 배포 하는 데 사용 됩니다.

-   **ISO 이미지 파일** -CD 또는 DVD에 배포 하거나 VM (가상 컴퓨터)에서 사용 하는 데 사용 됩니다. CD 또는 DVD를 굽는 대부분의 프로그램에는 해당 확장이 필요 하기 때문에 마법사에서 ISO 이미지에는 .iso 확장명을 사용 해야 합니다. 다른 위치를 지정 하지 않으면 이름 DaRT8와 함께 바탕 화면에 ISO 이미지가 만들어집니다.

-   **PowerShell 스크립트** – Dart 복구 이미지 마법사를 사용 하 여 선택할 수 있는 옵션을 기본적으로 제공 하는 명령으로 dart 복구 이미지를 만듭니다. 또한이 스크립트를 사용 하 여 DaRT 복구 이미지에서 파일을 추가 하거나 변경할 수 있습니다.

이 페이지에서 이미지 편집 확인란을 선택 하면 이미지 만들기 프로세스 중에 복구 이미지를 사용자 지정할 수 있습니다. 예를 들어 "winpeshl.ini" 파일을 변경 하 여 사용자 지정 시작 순서를 만들거나 타사 도구를 추가할 수 있습니다.

**만들 복구 이미지 파일의 형식을 선택 하려면**

1.  **이미지 만들기** 페이지에서 **찾아보기를** 클릭 하 여 이미지 파일의 출력 폴더를 선택 합니다.

    **참고**  이미지 크기는 사용자가 선택 하는 도구와 마법사에서 추가 하는 파일에 따라 달라 집니다.

     

2.  **이미지 이름** 상자에 DaRT 복구 이미지의 이름을 입력 하거나 기본 이름 (DaRT8)을 그대로 사용 합니다.

    마법사에서이 이름으로 출력 경로에 하위 폴더를 만듭니다.

3.  만들려는 이미지 파일 형식을 선택 합니다.

4.  다음 중 하나를 선택 합니다.

    -   이미지 파일을 만들기 전에 복구 이미지의 파일을 변경 하려면 **이미지 편집** 확인란을 선택 하 고 **준비**를 클릭 합니다.

    -   파일을 변경 하지 않고 복구 이미지를 만들려면 **만들기**를 클릭 합니다.

5.  

    **다음**을 클릭합니다.

## 복구 이미지 파일 편집


이미지 만들기 페이지의 이미지 편집 확인란을 선택한 경우에만 복구 이미지를 편집할 수 있습니다. 복구 이미지를 편집할 준비가 된 후에는 부팅 가능 미디어를 만들기 전에 복구 이미지 파일을 추가 하 고 수정할 수 있습니다. 예를 들어 시작에 대 한 사용자 지정 순서를 만들고 다양 한 타사 도구를 추가할 수 있습니다.

**복구 이미지 파일을 편집 하려면**

1.  **이미지 편집** 페이지에서 Windows 탐색기에서 **열기** 를 클릭 합니다.

2.  대화 상자에 나열 된 폴더에 하위 폴더를 만듭니다.

3.  원하는 파일을 새 하위 폴더에 복사 하거나 필요 하지 않은 파일을 제거 합니다.

4.  **만들기** 를 클릭 하 여 복구 이미지 만들기를 시작 합니다.

## 복구 이미지 파일 생성


파일 생성 페이지에서 이미지 만들기 페이지에서 선택한 파일 형식에 대 한 DaRT 복구 이미지가 생성 됩니다.

**복구 이미지 파일을 생성 하려면**

-   **파일 생성** 페이지에서 **다음** 을 클릭 하 여 복구 이미지 파일을 생성 합니다.

## 복구 이미지를 CD, DVD 또는 USB에 복사


부팅 가능 미디어 만들기 페이지에서 선택적으로 이미지 파일을 CD, DVD 또는 UFD (USB 플래시 드라이브)에 복사할 수 있습니다. 마법사를 다시 시작 하 여이 페이지에서 부팅 가능한 미디어를 추가로 만들 수도 있습니다.

**참고**  PXE (부팅 전 실행 환경) 및 로컬 이미지 배포는 System Center Configuration Manager 서버 및 Microsoft 개발 도구 키트와 같은 추가 엔터프라이즈 도구를 필요로 하므로 기본적으로이 도구를 통해 지원 되지 않습니다.

 

**복구 이미지를 CD, DVD 또는 USB에 복사 하려면**

1.  **부팅 가능 미디어 만들기** 페이지에서 복사 하려는 iso 파일을 선택 합니다.

2.  CD, DVD 또는 USB를 삽입 한 다음 드라이브를 선택 합니다.

    **참고**  드라이브가 인식 되지 않고 새 드라이브를 설치 하는 경우 **새로 고침** 을 클릭 하 여 마법사에서 사용 가능한 드라이브 목록을 강제로 업데이트 하도록 할 수 있습니다.

     

3.  **부팅 가능 미디어 만들기** 단추를 클릭 합니다.

4.  다른 복구 이미지를 만들려면 다시 시작을 클릭 하거나 원하는 모든 미디어 만들기가 완료 되 면 **닫기를** 클릭 합니다.
