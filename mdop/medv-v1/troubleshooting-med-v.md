# MED-V 문제 해결


이 섹션에서는 MED-V (Microsoft 엔터프라이즈 데스크톱 가상화)의 일반적인 문제를 해결 하는 데 도움이 되는 정보를 제공 합니다.

## 호스트 해상도를 변경한 다음 MED-V 작업 영역을 최대화 하면 데스크톱이 검게 표시 됩니다.


전체 데스크톱 모드에서 작업 하는 경우 호스트 해상도를 변경 하 고 MED-V 작업 영역 창을 최대화 하면 바탕 화면이 검은색으로 나타나고 MED-V 작업 영역이 응답 하지 않을 수 있습니다.

### 해결 방법

MED-V 작업 영역을 중지 한 다음 시작 합니다.

## 네트워크 어댑터가 사용 안 함으로 설정 된 상태에서 MED-V 작업 영역을 시작 하 고 나중에 어댑터를 사용 하도록 설정 하면 네트워크 연결이 복원 되지 않음


브리지 모드에서 MED-V 작업 영역을 구성한 다음 네트워크 어댑터를 사용 하지 않도록 설정 하는 동안 MED-V 작업 영역을 시작 하는 경우 어댑터가 나중에 사용 되는 경우 해당 어댑터를 통한 네트워크 연결이 복원 되지 않습니다.

### 해결 방법

MED-V 작업 영역을 중지 한 다음 시작 합니다.

## 이미지는 컴퓨터 당 하나의 Windows 사용자만 사용할 수 있습니다.


MED-V 작업 영역 이미지는 이미지를 다운로드 하거나 가져온 Windows 사용자만 사용할 수 있습니다. 이 사용자는 다운로드 한 이미지가 있는 폴더에 대 한 사용 권한이 있는 관리자 외에는 사용자만이 사용할 수 있습니다.

### 해결 방법

이미지 저장소의 ACL (액세스 제어 목록)을 수동으로 변경 합니다.

## 사용자 권한이 있는 Configuration Manager를 사용 하 여 MED-V를 설치 하는 경우 제거에 실패 함


Microsoft System Center Configuration Manager를 사용 하 여 MED-V를 설치 하 고 패키지의 실행 모드를 사용자 권한으로 설정한 경우에는 관리자만 MED-V를 제거할 수 있다는 오류 메시지와 함께 제거에 실패 합니다.

### 해결 방법

MED-V에 대 한 Configuration Manager 패키지를 만들 때 실행 모드를 관리 권한으로 설정 합니다.

## 설치 후 클라이언트를 실행 하도록 설치 되어 있는 회사 배포 시스템을 사용 하 여 MED-V를 설치 하는 경우 클라이언트를 실행할 수 없습니다.


MED-V 클라이언트를 회사 배포 시스템을 사용 하 여 설치 하 고 설치 패키지는 클라이언트를 시스템 계정으로 실행 한 후에 클라이언트가 실행 되 고 있음을 알 수 없으며 (알림 영역을 제외 하 고),이를 사용 하 여 대화형 작업을 할 수 없습니다.

### 해결 방법

회사 배포 시스템을 사용 하 여 MED-V를 설치 하는 경우 *시작 \ _MEDV = 0* .msi 매개 변수를 사용 합니다.

## MED-V 테스트 이미지가 시작 되지 않음


MED-V 테스트 이미지를 시작 하지 못하면 복구 되지 않으며 "GINA를 로드 하지 못했습니다." 오류 메시지와 함께 모든 이후 시작이 실패 합니다.

### 해결 방법

기존 테스트 이미지를 삭제 하 고 다시 만듭니다.

## 잘못 된 자격 증명을 사용 하 여 도메인에 가입 하려고 시도 하면 이미지가 도메인 가입에 성공 하지 않습니다.


가상 컴퓨터 첫 설치 스크립트의 일부인 참가 도메인 빌드 블록에 구성 오류가 있는 경우 도메인에 가입 하려고 할 때 MED-V 작업 영역이 실패 합니다. 구성 오류가 복구 된 후 MED-V 작업 영역에 포함 된 이미지가 도메인에 참가할 수 없습니다.

### 해결 방법

이미지가 배포 된 경우 이미지를 다시 배포 합니다. 이미지가 테스트 이미지인 경우 이미지를 다시 만듭니다.

## MED-V는 여러 모니터를 지원 하지 않습니다.


MED-V는 여러 모니터에서 게시 된 응용 프로그램을 표시 하는 것을 지원 하지 않습니다. 게시 된 응용 프로그램 및 기타 클라이언트 창은 잘못 된 화면에 표시 될 수 있으며 화면 연결이 끊어지면 MED-V가 모니터에 화면을 보내려고 시도 하 여 연결 된 모니터에 아무 것도 표시 되지 않습니다.

### 해결 방법

추가 화면의 연결을 끊고 클라이언트를 다시 시작 합니다.

## MED-V 작업 영역 시작 중에 호스트가 충돌 하는 경우 MED-V 작업 영역이 시작 되지 않을 수 있음


MED-V 작업 영역 시작 프로세스가 진행 되는 동안 호스트가 중단 되 고 "루트 요소 누락 됨" 이라는 오류 메시지가 표시 되는 경우 MED-V 작업 영역이 빈 가상 머신 구성 (VMC) 파일에 데이터를 추가할 수 있으며,이 경우 시작 프로세스에 오류가 발생 합니다.

### 해결 방법

빈 VMC 파일을 기본 이미지의 VMC 파일로 바꿉니다.

## 키보드는 게시 된 응용 프로그램 창에서 응답 하지 않습니다.


MED-V 작업 영역에서 게시 된 응용 프로그램이 포커스를 할 때 Windows 로고 키를 누르면 키보드가 게시 된 응용 프로그램 창에서 더 이상 응답 하지 않습니다.

### 해결 방법

게시 된 응용 프로그램이 포커스를 둔 상태에서 Windows 로고 키를 누릅니다.

## 도메인 MED-V 작업 영역이 도메인 자격 증명을 업데이트 하지 않음


도메인 환경에서 영구 MED-V 작업 영역을 사용 하는 경우 도메인 암호를 변경 하는 경우 MED-V 클라이언트가 MED-V 작업 영역 도메인 자격 증명을 업데이트 하지 않습니다. 게시 된 응용 프로그램이 네트워크 리소스에 액세스 하려고 하면 자격 증명이 만료 되었음을 알리는 오류 메시지가 표시 됩니다.

### 해결 방법

MED-V 작업 영역 운영 체제를 다시 시작 합니다.

## 게시 된 최대화 응용 프로그램 windows 커버 호스트 작업 표시줄


게시 된 응용 프로그램 창을 전체 화면으로 최대화 하면 호스트 작업 표시줄에 포함 될 수 있습니다.

### 해결 방법

다음 중 하나를 수행합니다.

게시 된 응용 프로그램 창을 최소화 하 여 알림 영역에 대 한 액세스를 얻고 MED-V 작업 영역을 다시 시작 합니다.

게시 된 응용 프로그램 창을 최소화 하 고 창을 최대화 상태로 복원 합니다.

## MED-V 서버 구성 관리자에서 사용자 또는 그룹 추가가 작동 하지 않음


사용자 또는 그룹 **선택** 대화 상자에서 사용자 또는 그룹을 추가 하는 경우, Med-v 서버 구성 관리자의 액세스 제어 목록에 선택한 사용자 또는 그룹이 추가 되지 않습니다.

### 해결 방법

**사용자 또는 그룹 이름 입력** 대화 상자를 사용 하 여 사용자 또는 그룹을 추가 합니다. 자세한 내용은 [Med-v 서버 구성 요소를 설치 하 고 구성 하는 방법을](how-to-install-and-configure-the-med-v-server-component.md#bkmk-configuringpermissions)참조 하세요.

## MED-V는 windows 7 용 Windows Virtual PC가 설치 된 컴퓨터에서 작동 하지 않습니다.


MED-V에는 Windows 가상 PC2007 필요 합니다. Windows7 및 Virtual PC2007 SP1 용 Windows 가상 PC를 같은 컴퓨터에 설치할 수 없습니다.

### 해결 방법

Virtual PC2007 SP1 및 MED-V를 설치 하기 전에 Windows7의 가상 PC를 제거 합니다.

## <a href="" id="med-v-does-not-support-virtual-pc-and-windows-xp-mode-images-"></a>MED-V는 가상 PC 및 WindowsXP 모드 이미지를 지원 하지 않습니다.


MED-V 1.0 SP1은 Windows7 용 Windows 가상 PC에서 만든 이미지를 지원 하지 않습니다. Windows7 image 용 가상 PC를 사용 하는 경우 시작 중에 클라이언트가 실패 합니다.

### 해결 방법

Virtual PC2007 SP1을 사용 하 여 MED-V 이미지를 만듭니다.

## Windows 방화벽은 가상 PC2007 SP1 네트워크 활동을 차단 합니다.


기본적으로 Windows 방화벽은 가상 PC2007 SP1 네트워크 활동을 차단 하 고, 클라이언트 컴퓨터에서 가상 PC2007 SP1이 시작 되 면 해당 시작 순서와 모든 네트워크 액세스를 차단 하는 방화벽 메시지가 있습니다.

### 해결 방법

마지막 사용자가 MED-V를 사용 하기 전에 그룹 정책을 사용 하 여 방화벽 예외를 업데이트 합니다.

## 클라이언트를 업그레이드할 때 오류 메시지가 표시 되는 경우


MED-V 1.0에서 MED-V 1.0 SP1으로 클라이언트를 업그레이드 하는 경우 MED-V 작업 영역이 정의 되지 않았음을 알리는 메시지가 표시 될 수 있습니다.

### 해결 방법

클라이언트를 닫고 다시 시작 합니다.
